# r2-git2

A native macOS **menu bar** app — a GitHub Swiss-army knife. A compact dark popover that gives
you fast access to your pull requests, GitHub Actions, and mentions, without replacing GitHub
itself: every item opens the matching GitHub URL in your browser.

Built with Swift + SwiftUI (`MenuBarExtra`, `.window` style), targeting **macOS 15 (Sequoia)+**.

## Features

- **Auto-discovery** of every repo you can access (personal, collaborator, and org repos). Pick
  which to *follow* in Settings — **nothing is followed by default**.
- **Pull Requests**, in two groups:
  - **PRs for me** — where you are review-requested, assigned, or the author (any repo,
    deduplicated across roles), each tagged with its role and CI status.
  - **Open PRs** on your followed repos.
- **GitHub Actions** per followed repo: in-progress/queued runs on all branches, plus **recent
  failures** (successful runs are hidden to cut noise; window is `Config.recentFailureWindow`,
  default 6 h) — with workflow name, branch, colored status, duration, and relative time.
- **Notifications & mentions** from your GitHub inbox, mentions highlighted.
- **Auto-refresh** every 10 min (configurable) + a manual refresh button; "Updated X ago" header.
- **Conditional requests** (`ETag` / `If-None-Match`): a `304 Not Modified` doesn't consume your
  rate limit. Remaining quota is shown in Settings → Preferences (debug).
- **Native notifications** for: a new PR requesting your review / assigned to you, a new mention,
  or a failed run on a followed repo — deduplicated so you're not re-notified on the next refresh.
- **Menu-bar badge**: count of PRs to review, and an alert glyph when a run recently failed.
- **Open at login** toggle (`SMAppService`).
- **Auto-update via Sparkle** (EdDSA-signed, no Apple Developer ID required).
- Token stored in the **macOS Keychain** — never on disk or in `UserDefaults`.

## Project layout

```
Sources/
  App/          R2Git2App (@main MenuBarExtra), AppDelegate, Config, Info.plist, entitlements
  Models/       Codable GitHub DTOs (GitHubModels) + UI domain models (DomainModels)
  Networking/   GitHubClient (REST/Search/GraphQL), GitHubEndpoint, ETagCache, APIError
  Auth/         KeychainStore
  Store/        AppStore (@Observable orchestrator), AppSettings, NotificationManager
  Updater/      UpdaterViewModel (Sparkle)
  Views/        PopoverView, LoginView, SettingsView, MenuBarLabel, Sections/, Components/
Tests/          ETag cache + model-decoding / domain-logic unit tests
.github/workflows/  ci.yml, release.yml
project.yml     XcodeGen spec (the .xcodeproj is generated, not committed)
```

**Design notes**
- The project is generated by **XcodeGen** from `project.yml`; the `.xcodeproj` is git-ignored.
- The app is **not sandboxed** (simplest for Sparkle + Keychain outside the App Store); Hardened
  Runtime is on so notarization can be added later without restructuring.
- Networking base URLs live in `Config`/`GitHubEndpoint` so a **GitHub Enterprise** host is a
  one-line change.
- PRs are fetched via **GraphQL** so each PR arrives *with* its CI `statusCheckRollup` in a single
  request (a big rate-limit win); everything else uses REST.

## Setup

### 1. Authentication — Personal Access Token (classic)

The app authenticates with a **classic Personal Access Token** you generate yourself. This avoids
the OAuth per-organization "grant" consent screen while keeping full API access (cross-repo PR
search, the notifications inbox, and org repos). No Client ID or client secret is needed.

You generate the token from the app itself on first launch (see step 3), or ahead of time at
**GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate
new token (classic)** with scopes:

- `repo` — PRs & Actions on private + public repos
- `read:org` — list your organizations
- `notifications` — read the notifications inbox (mentions)

> If an organization enforces **SAML SSO**, click **Configure SSO → Authorize** next to the token
> once so it can read that org's private resources.

### 2. Build locally

Requires Xcode 16+ and [XcodeGen](https://github.com/yonaskolb/XcodeGen):

```bash
brew install xcodegen
xcodegen generate          # creates r2-git2.xcodeproj
open r2-git2.xcodeproj     # ⌘R to run
```

Or from the command line:

```bash
xcodebuild -project r2-git2.xcodeproj -scheme r2-git2 -destination 'platform=macOS' build
xcodebuild -project r2-git2.xcodeproj -scheme r2-git2 -destination 'platform=macOS' test
```

### 3. Sign in

On first launch the popover shows a sign-in screen:

1. Click **Create a token on GitHub** — this opens GitHub's "new classic token" page with the
   three required scopes **pre-selected**. Choose an expiry (or *No expiration*), generate, and
   copy the token (`ghp_…`).
2. Paste it into the app and click **Save token**. It's validated (`GET /user`) and stored in the
   macOS **Keychain** — never on disk or in `UserDefaults`.

Use **Settings → Account → Sign out** to purge the token from the Keychain.

## Distribution & auto-updates (Sparkle, no Apple Developer ID)

Updates are signed with a free **EdDSA** key pair — no paid Apple Developer ID or notarization is
required to ship.

### 1. Generate the Sparkle key pair (once)

Download the Sparkle tools (matching `SPARKLE_VERSION` in `release.yml`) and run:

```bash
curl -fsSL -o sparkle.tar.xz \
  https://github.com/sparkle-project/Sparkle/releases/download/2.6.4/Sparkle-2.6.4.tar.xz
tar -xf sparkle.tar.xz
./bin/generate_keys
```

This prints a **public key** and stores the **private key** in your login Keychain. To export the
private key for CI:

```bash
./bin/generate_keys -x sparkle_private_key.txt   # writes the private key to a file
```

### 2. Wire the keys in

- Put the **public key** into `project.yml` → `SUPublicEDKey` (replace
  `REPLACE_WITH_SPARKLE_PUBLIC_ED_KEY`), then re-run `xcodegen generate`.
- Add the **private key** as a GitHub Actions secret named **`SPARKLE_PRIVATE_KEY`**
  (repo → Settings → Secrets and variables → Actions). Paste the contents of
  `sparkle_private_key.txt`.

The feed URL is already set in `project.yml`:
`https://github.com/FabienVINCENT/r2-git2/releases/latest/download/appcast.xml` — Sparkle always
pulls the appcast attached to the latest Release.

### 3. Cut a release

```bash
git tag v1.0.0
git push origin v1.0.0
```

The **Release** workflow then:
1. builds the Release `.app` (version derived from the tag),
2. packages a clean `.dmg` (app + drag-to-`/Applications`) via `hdiutil`,
3. signs the `.dmg` with your EdDSA key (`sign_update`),
4. generates `appcast.xml` with `sparkle:edSignature` and file length,
5. creates a **GitHub Release** attaching the `.dmg` and `appcast.xml`.

### Gatekeeper note ⚠️

Because the app is **not** signed with an Apple Developer ID or notarized, the first launch (and
each update) triggers a Gatekeeper warning. To open it:

**System Settings → Privacy & Security → scroll to the r2-git2 message → "Open Anyway"**
(or right-click the app → **Open** → **Open**).

### TODO: Apple notarization (later)

If you get an Apple **Developer ID**, Gatekeeper warnings disappear. The hooks are already in
place:

- `project.yml` keeps **Hardened Runtime** enabled and has an `entitlements` file ready for the
  sandbox/network entitlements notarization expects.
- `release.yml` has a marked spot in the **Build Release .app** step — swap `CODE_SIGNING_ALLOWED=NO`
  for a real identity, then add:
  ```bash
  codesign --deep --force --options runtime --sign "Developer ID Application: …" r2-git2.app
  xcrun notarytool submit r2-git2.dmg --apple-id … --team-id … --password … --wait
  xcrun stapler staple r2-git2.dmg
  ```
  Add the corresponding Apple secrets (`APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_APP_PASSWORD`, signing
  certificate) alongside `SPARKLE_PRIVATE_KEY`.

## CI

- **`ci.yml`** — on every PR / push to `main`: generates the project, builds, and runs tests.
- **`release.yml`** — on `v*` tags: build → dmg → EdDSA-sign → appcast → GitHub Release.

**Required secrets**

| Secret | Used by | Purpose |
| --- | --- | --- |
| `SPARKLE_PRIVATE_KEY` | `release.yml` | EdDSA private key to sign updates |
| *(later)* `APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_APP_PASSWORD` | `release.yml` | Apple notarization |
